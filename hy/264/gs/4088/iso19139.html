Abstract:

The Satellite View of the Conterminous United States, with Shaded Relief map layer is a 200-meter-resolution simulated-natural-color image of the United States, delivered in the Albers Equal-Area Conic map projection. Vegetation is generally green, with forests in darker green and grasslands or shrublands in lighter green. Areas of high reflectance, including urban areas, rock, and dry bare soil, are shown in shades of pink. Very bright areas, such as snow and ice, are colored blue. The image was produced by combining Landsat Thematic Mapper (TM) imagery from the Landsat 4 and Landsat 5 satellites with relief enhanced by shading. This map layer was previously distributed as Satellite View of the Conterminous United States. This layer is part of the 1997-2014 edition National Atlas of the United States.

Purpose:

The Satellite View with Shaded Relief image was developed to portray the terrain and land cover of the conterminous United States. The image is intended for visual purposes only. The original Landsat Thematic Mapper imagery and National Elevation Dataset (NED) data must be used for conducting analysis.

Process Step:

A 4-compact-disc set of Thematic Mapper (TM) imagery titled "United States of America Digital Landsat Mosaics" was the basis for creating the Satellite View of the Conterminous United States, with Shaded Relief. Earth Satellite Corporation, under contract to NASA and the USGS, processed the data. USGS and NASA distribute the data for unrestricted non-commercial use. The average acquisition date for the Landsat Orthorectified TM data is 1990 (+/- three years). To attain cloud-free coverage, there are a small number of patches taken from scenes from as early as 1985 and from as late as 1996. Images on the CDs are stored at 28.5-meter resolution in a wavelet- compressed format called MrSid. All image data on CD are in UTM projection and WGS84 datum. Each image covers the width of a UTM zone, or 6 degrees of longitude, and is 5 degrees of latitude high. Images were opened in Adobe Photoshop, with an 8x expansion factor to produce 228-meter-resolution images, and were saved as TIFF's. TIFF world files (.tfw) were created to accompany each image extracted so it could be loaded into ArcInfo Grid for further processing. Images were assembled in strips by UTM zone using the ArcInfo Grid command Merge. For the conterminous United States, there were 11 UTM zones, numbers 10 through 20. Zero values were set to NULL for each image, so as to not influence any resampling during reprojection. Each image was reprojected from UTM to a Lambert Azimuthal Equal-Area projection (LAZ) with a center of 100 degrees west, 45 degrees north, and an output pixel size of 200 meters. The datum for the source TM data is WGS84. For 200-meter resolution image data, there is no significant difference between WGS84 and NAD83, so the images were redefined as NAD83 to match other National Atlas map layers. The LAZ images for each zone were mosaicked in ArcInfo. The image was exported from ArcInfo using the command GRIDIMAGE. The LAZ image was opened in Photoshop and the two masks applied to set ocean water and non-U.S. land to standard colors required by the National Atlas online Map Maker. Ocean water was set to RGB value (132, 193, 255) and non-U.S. land was set to RGB value (132, 158, 172).

Date And Time: 2004-10-01T00:00:00

Process Step:

Elevation data for the conterminous United States were extracted from the NED, brought into ArcInfo, and projected to a Lambert Azimuthal Equal-Area projection centered on 100 degrees West, 45 degrees North, with a pixel size of 200 meters. The land/water mask was applied to the 16-bit elevation grid, which clipped the grid to the National Atlas shoreline. The grid was exported from ArcInfo to create the TIFF image. The masked elevation grid is available on request. More information on the NED may be found by following the links at <http://ned.usgs.gov>.

Date And Time: 2005-05-01T00:00:00

Process Step:

The Satellite View of the Conterminous United States, with Shaded Relief was created by combining two files: The first file, l48clip.tif, was opened in Adobe Photoshop. Some of the more obvious seam lines, which were the result of the mosaicking process, were lessened by creating selections on either side of the seam and applying Photoshop's Adjust Curves command to try and match the appearance across the seam. Once seam retouching was completed, the image file was converted from RGB format to CMY format (no black generated). Color balancing, tonal enhancements, and image sharpening procedures were performed on the CMY image. The second file, l48_laz_200m_dem.tif, was imported into ArcGrid using the ArcInfo command IMAGEGRID. Since IMAGEGRID does not support the conversion of signed image data, the ArcInfo command CON was performed on the grid to restore negative values. A grayscale shaded relief image of the elevation data was created using the GRID command HILLSHADE, then exported to a TIFF image. In Adobe Photoshop, the grayscale shaded relief image was added to the Landsat CMY image as a black (K) band to create a CMYK Landsat image with relief shading. Overall contrast, tonal balance, and image sharpness were adjusted. The image was converted from CMYK format to RGB format. The conversion from RGB to CMYK and back to RGB changed the values for water and non-U.S. land that had been set in the source images. The masks to set values for ocean water and non-U.S. land were reapplied to the image in Adobe Photoshop. The RGB image was ingested into ArcInfo's Grid module, saved as a TIFF from ArcInfo, and processed through a USGS GeoTIFF software program to create a fully-compliant GeoTIFF with valid header information. For the Satellite View of the Conterminous United States, with Shaded Relief data, the datum is NAD83. Because there is no ellipsoidal form of the Lambert Azimuthal Equal-Area projection (LAZ), once data have been projected to LAZ they lose any datum affiliation and instead are based on a sphere with a radius of 6370997 meters. However, some commonly-used software packages will report an error if a datum is not specified, so to accommodate such software, the GeoTIFF of this map layer retains the NAD83 datum.
