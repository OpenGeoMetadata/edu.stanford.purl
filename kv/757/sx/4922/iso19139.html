Process Step:

For information on the biosampling process, please see the California Salmonid Stream Habitat Restoration Manual (www.dfg.ca.gov/fishing/manual3.pdf). The biosampling data was entered into a Dbase file (biosampl.dbf) using the DOS-based Habitat 8.3 program in 1998-2000. The biosampl.dbf file was imported into ArcView 3.2a. The layer was converted into a linear shapefile using dynamic segmentation along a 1:100,000 scale routed streams layer (russ-routed). The "From" distance field was calculated by converting the "Dist" field (distance from confluence for start of the survey) from feet to meters; the"To" field was calculated by adding the sample distance in meters (Sampdistm) to the "From" field. The sample distance in meters was calculated by converting the "Sampl_dist" field from feet to meters. Without proper calibration of most of the streams, starting and ending locations of the linear survey features are only approximate (probably within 200 meters). Where no sample distance was recorded but a starting distance was, a distance of 1-meter was entered so a linear feature could be created. Where no start of survey distance was recorded ("dist"), no attempt was made to do the dynamic segmentation of the biosampling record. Once a linear feature was created, a points layer was created by using the "Shapes to centroids" function of the "Xtools" ArcView 3.2 extension. This means that the point feature represents the approximate linear center of the survey, not its start or end, except where no sample distance was entered, and the point represents the start of survey. Some exceptions exist where points where found to have been dynamically segmented along the wrong creek due to incorrect primary name code (Pnmcd) stream identifiers.
