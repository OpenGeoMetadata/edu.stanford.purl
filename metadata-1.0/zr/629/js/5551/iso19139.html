Abstract:

This polygon shapefile contains primary health care professional shortage areas (HPSAs) in California. On January 26, 2004, the California Healthcare Workforce Policy Commission (Commission) formally adopted a means to create the PCSA map. The process for identifying PCSAs uses the rule base listed below. In March, 2015, the OSHPD staff presented the Commission with information suggesting an update to the map to include current data on (1) physicians, (2) poverty and (3) population. Percent below Poverty Level (100%) Value Range Weighted Score 5.0 or Less 0 5.1 - 10.0 1 10.1 - 15.0 2 15.1 - 20.0 3 20.1 - 25.0 4 25.1 or Greater 5 (maximum) Physician-to-Population Ratio Value Range Weighted Score Lower than 1:1,000 0 1:1,000 to 1:1,500 1 1:1,500 to 1:2,000 2 1:2,000 to 1:2,500 3 1:2,500 to 1:3,000 4 Higher than 1:3,000 5 (maximum) * Any MSSA with a score of 5 or greater is defined as a PCSA. PCSAs are used as a means to help the Commission rank applications based on the number of program graduates and training sites inside areas of unmet need. PCSAs are the only consistently applied rule base to defining shortages of physicians, as the other designations are applicant based and require prior knowledge that a shortage might exist. This data is aggregated by Medical Service Study Area (MSSA) to obtain a count of primary care physicians by MSSA. Primary Care Shortage Areas are updated Annually and are used in the Song-Brown Grant Program for Family Medicine, Family Nurse Practitioner-Physician Assistant and Primary Care Residency programs. This update to data for the PCSA was approved by the California Healthcare Workforce Policy Commission on March 4, 2015. This layer is part of the Healthcare Atlas of California.

Purpose:

This data for Primary Care Shortage Area (PCSA) was developed by the Office of Statewide Health Planning and Development's (OSHPD) Healthcare Workforce Development Division (HWDD). The data is used to support the following programatic areas: 1) encourage demographically underrepresented groups to pursue healthcare careers 2) identifies geographic areas of unmet need, and 3) encourages primary care physicians and non-physician practitioners to provide healthcare in medically underserved areas in California.

Lineage:

Process Step:

GP Model 1 - Task Preparation. Template feature classes were created from previous MSSA Layer (in this case 2007-9) and a new layer was created that used this template. Field values not affected by this update were appended. The Task Preparation model creates a new file geodatabase containing: a. New template-based MSSA and Detail feature classes with geometry and fields not updated, but appended b. One Claritas table used in the demographic field updates At 2010, this model has been updated to be a more generic process, where standard feature classes with the standard names MSSA and MSSA_Detail are created in a file geodatabase (also created by the model). This helps standardize the running of the next model, Calculate Fields which uses a lot of calculation expressions which otherwise need to be modified every year it is run, by virtue of it previously having used the specific year in the parameter names. For example, [MSSA2007.PCT_AMIND] is now a standard [MSSA.PCT_AMIND] which can be used for any year from now on.

Process Step:

GP Model 2 - Calculate Fields The Calculate Fields model updates the fields by: (a) Joining population, race, ethnicity and age table (SPFCAT06) to Detail feature class (b) Calculating population, race, ethnicity and age count, percentage and ratio fields in Detail feature class and removing join (c) Joining Census_Poverty_2000 (Poverty2000v3.mdb) to Detail feature class (d) Calculating percent poverty fields in Detail feature class and removing join (e) Calculating population in poverty fields in Detail feature class (f) Creating summary table with sums of count fields in the Detail feature class by MSSA_ID and joining to MSSA feature class (g) Calculating population, poverty, race, ethnicity and age count, percentage and ratio fields and population density field in the MSSA feature class and removing join (h) Creating a frequency table by MSSA_ID of Physicians in 20110506_InfoUSA_Geocode.gdb and joining to MSSA feature class. The Physicians from the InfoUSA2010 data were first filtered to include only those specific to MSSA classifications. The classifications are: General Practice, Family Practice, Internal Medicine, Pediatrics, Geriatrics, Obstetrics, Gynecology, Obstetrics & Gynecology and Geriatrics Family Practice. (i) Calculating primary care physician counts in MSSA feature class and removing join (j) Creating a frequency table by MSSA_ID of Dentists in 20110506_InfoUSA_Geocode.gdb and joining to MSSA feature class (k) Calculating dentist counts in MSSA feature class and removing join (l) Creating a frequency table by MSSA_ID of Psychiatrists in 20110506_InfoUSA_Geocode.gdb and joining to MSSA feature class (m) Calculating Psychiatrist counts in MSSA feature class and removing join (n) Calculating practitioner ratios (o) Updating Civilian Population PCSA Flag and Civilian Population PCSA Weighted Score fields using PCSA 2009 data. (p) Emptying (making null) Primary Care Shortage Area (PCSA) Flag and PCSA Weighted Score fields.
