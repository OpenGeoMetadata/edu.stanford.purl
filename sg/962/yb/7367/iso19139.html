Abstract:

This layer is a georeferenced raster image displaying elevation data for Alaska using an Albers projection. The elevation data were derived from National Elevation Dataset (NED) and show the terrain at a resolution of 100 meters. The NED is a raster product assembled by the U.S. Geological Survey, designed to provide national elevation data in a seamless form with a consistent datum, elevation unit, and projection. Data corrections made in the NED assembly process minimize artifacts, permit edge matching, and fill sliver areas of missing data. The NED is updated on a bi-monthly basis. This data set was derived from NED data released in December, 2010, This layer is part of the 1997-2014 edition National Atlas of the United States.

Purpose:

The 100-meter resolution elevation data were developed to portray the terrain of the United States at 1:1,000,000 scale. They are intended primarily for visual purposes. The original NED data should be used for conducting analysis and determining the most accurate elevation values. No responsibility is assumed by the National Atlas of the United States in the use of these data.

Process Step:

Four elevation mosaics were extracted from the National Elevation Dataset and used to produce elevation data files of the conterminous United States, Alaska, Hawaii, and Puerto Rico and the U.S. Virgin Islands. This map layer includes the data file for Alaska. Elevations in NED are stored in NAD83 geographic coordinates as 32-bit signed floating point data at 1-arc-second resolution. Extracted data were converted to 16-bit signed integer, projected to the NAD83 Albers Equal-Area Conic projection with a pixel size of 100 meters, and the data extent was set to match the National Atlas neatline extent for the subject area. The Alaska data set includes elevation data only for areas in the United States. Using the ArcGIS Feature-to-Raster command, a 0/1 land/water mask for Alaska was created from the 1:1,000,000-scale National Atlas coastline, at the same 100-meter resolution and with the same extent as the elevation data. The land/water mask was applied to the elevation grid with a software routine that used the ArcGIS Multiply command to clip elevation values that fell in ocean areas. The routine used the ArcGIS Nibble command to fill out elevation values to the coastline. Ocean and non-U.S. areas are set to an elevation value of 0. An intermediate product that has these values set to Null (or Nodata) is available on request. The land/water mask is also available on request.
