Abstract:

This line shapefile portrays the current routes used for conducting Breeding Bird Surveys in the lower 48 States. This layer is part of the 1997-2014 edition of the National Atlas of the United States.

Purpose:

The digital data sets are used to create electronic and hard copy maps of the breeding bird survey routes. The data can be used to conduct geographic analysis on national and large regional scales. The data should be displayed and analyzed at scales appropriate for 1:100,000-scale data.

Measure Description:

ArcInfo software was used to create and maintain topological relationships between features. The ArcInfo command BUILD was run to ensure the topological consistency of the map layer. Each route was checked for missing and dangling arcs, correct length, proper direction, and correct route measures.

Completeness Omission:

Evaluation Method Description:

Lineage:

Process Step:

The Breeding Bird Survey route maps (BBSMAP) for most States were digitized with ArcView 3.0 and the US Streets Database (ESRIDA), which is based on the Tiger files. The arcs forming the routes were selected and shapefiles were created for each route. Routes in Idaho, Michigan, North Dakota, Ohio, West Virginia, Washington and part of Maryland were digitized using DeLorme's Street Atlas USA (DLRM). The files were converted to ArcInfo line coverages by State. BBS routes for Minnesota and Wisconsin were digitized by James M. Sales, University of Minnesota, Duluth, UMD Natural Resources Research Institute, Duluth, MN, 55812. The Virginia routes were provided to us by Jeffrey B. Trollinger, Fish and Wildlife Information System, Virginia Department of Game and Inland Fisheries, 4010 West Broad Street, Richmond, VA 23280. The California routes were provided to us by Allan Hollander, University of California, Santa Barbara, Santa Barbara, CA, 93106. The data was then converted into a format consistent with ours. We used ArcView to add newly created and missing routes to these files. For each State, an ArcInfo route system was added using the ArcInfo MEASUREROUTE command. The JOINITEM command was used to add the route name from the BBS Database (BBSDA). The APPEND command was then used to create a single coverage for the National Atlas.

Process Step:

The coverage was read into ArcInfo, converted to NAD 83, and converted from the Lambert projection to geographic coordinates. The resulting file was loaded into ArcView with the route system information as one shapefile and the breeding bird survey route arcs as another shapefile. To create a final file containing both the arc information and the route lengths, the tables for the two layers were joined using the rteno item and the two ID numbers from the route coverage were turned off. The resulting file was saved as a shapefile.
